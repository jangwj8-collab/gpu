예약 규칙
01예약은 반일 단위로만 가능합니다. 오전(09~13시) 또는 오후(13~18시) 중 선택하세요.
02같은 시간대·같은 GPU를 두 명이 예약할 수 없습니다. 먼저 예약한 사람이 우선입니다.
03본인이 등록한 예약(초록 블록)만 취소할 수 있습니다. 달력에서 블록을 클릭하세요.
04장기 독점 사용이 필요한 경우 팀원과 사전에 협의하세요.
05예약 후 30분 이내에 사용을 시작하지 않으면 다른 팀원이 사용할 수 있습니다.
설정 가이드 (GitHub Issues 연동)
Google Sheets 없이 GitHub만으로 예약 데이터를 관리합니다.
1
GitHub Personal Access Token 발급
GitHub → 우측 상단 프로필 → Settings → Developer settings
→ Personal access tokens → Tokens (classic) → Generate new token
→ 권한: repo 체크 → 생성 후 토큰 복사 (ghp_xxx...)
2
index.html 상단 설정값 입력
이 파일 스크립트 최상단의 두 줄을 수정하세요:
const GITHUB_TOKEN = 'ghp_여기에붙여넣기';
const GITHUB_REPO = '계정명/레포이름';
3
GitHub Issues 탭 활성화 확인
레포 → Settings → Features → Issues 체크 확인.
예약이 생성되면 Issues 탭에서 데이터를 직접 볼 수 있어요.
4
서버·GPU 정보 수정
스크립트 상단 const SERVERS = [...] 배열을 실제 서버 이름과 GPU 개수에 맞게 수정하세요.
⚠️ 토큰 보안 주의: GITHUB_TOKEN을 public 레포에 올리면 GitHub이 자동으로 무효화해요.
해결: 레포를 private으로 유지하거나, 토큰 권한을 issues only로 최소화하세요.
✅ 설정 완료 후: 팀원이 예약하면 GitHub Issues에 자동 저장되고, 달력이 실시간으로 동기화됩니다.
